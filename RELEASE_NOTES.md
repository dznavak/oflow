# Release Notes

## Recent Changes

- **e2e verification pipeline**: Added `scripts/e2e.sh` end-to-end test that validates the full oflow pipeline on every dev-workflow run — creates a real GitHub issue, spawns the daemon, and polls for a PR before allowing merge
- **Release notes workflow**: Added `release-notes` skill as a lightweight e2e test target that summarises recent git changes and opens a PR
- **Schema consolidation**: Merged 6 individual schema files into a single `src/state/schemas.ts`, reducing import complexity across the codebase
- **Bug fix — session stuck in loop**: Fixed a bug where completed sessions remained in the scheduler indefinitely after a 404 on label removal; `removeLabel` is now idempotent and `removeSession` is always called via `try/finally`
- **Bug fix — Zod validation**: Replaced unsafe `JSON.parse` cast in `readTaskContext` with strict `TaskContextSchema.parse(...)` for fast, clear failures on malformed state files
- **Bug fix — agent log reliability**: Fixed log writing to use direct fd stdio so output reaches disk even when the daemon event loop is paused; added separate `tail` process for terminal display
- **Skill path fixes**: Updated all dev-workflow step skill paths to the `agentskills.io` folder format (`.claude/skills/<name>/SKILL.md`) and replaced shell substitutions with `$OFLOW_CURRENT_TASK_ID`
- **Label-based task filtering**: Added `--label` option to `oflow board list`, `oflow board pick`, and `oflow run` for filtering tasks by a custom label
- **Daemon logging improvements**: Daemon now logs board/agent/label/slots config on start, shows PID + log path on spawn, duration on completion, and log path on failure
- **Test fixture deduplication**: Extracted shared `makeTask()`, `makeMockAdapter()`, and `makeMockStateManager()` helpers into `src/test-utils.ts`, eliminating copy-paste across test files

_Generated by oflow e2e verification._
