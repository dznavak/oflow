# Release Notes

## Recent Changes

- **Schema consolidation**: Merged 6 individual schema files into a single `src/state/schemas.ts` with proper Zod validation, simplifying imports across the codebase.
- **Session loop fix**: Fixed daemon sessions getting stuck in an infinite loop when the `oflow-in-progress` label had already been removed — label removal is now idempotent and sessions are always cleaned up.
- **Test fixture refactor**: Extracted duplicate test helpers (`makeTask`, `makeMockAdapter`, `makeMockStateManager`) into a shared `src/test-utils.ts`, eliminating copy-paste across test files.
- **Reliability improvements**: Added Zod validation for `readTaskContext`, fixed agent log writing using direct fd stdio to prevent data loss, and corrected dev-workflow step skill paths.
- **E2E verification**: Added `scripts/e2e.sh` and a `release-notes` workflow skill as the e2e test target — the full pipeline (issue → daemon → PR) is now validated on every dev-workflow run.
- **Daemon logging**: Fixed a bug where all active sessions were logged as "started" on each poll; now only newly started sessions are logged.
- **Claude spawn & log streaming**: Spawn `claude` via stdin to avoid YAML frontmatter parsing issues; pipe agent output to both log file and terminal in real-time with task prefix.
- **Label-based task filtering**: Added `--label` flag to `oflow board list`, `board pick`, and `run` commands for filtering tasks by custom labels.
- **Skills restructure**: Migrated skills to `.claude/skills/` folder format with `SKILL.md` per skill, matching the agentskills.io open format so Claude Code discovers them natively.
