# Release Notes

## Recent Changes

- **Per-step timeout and hang detection**: Daemon loop now kills steps that exceed a configurable timeout (default 900s), marks them as timed-out, and releases the session slot so the daemon continues
- **Event-driven waitForCompletion**: Replaced `setInterval` polling in ClaudeCodeAdapter and OpencodeAdapter with direct promise resolution in the child process `close` event, eliminating potential hangs
- **GitLab board adapter**: Full `BoardAdapter` implementation for GitLab Issues supporting label-based workflow management and task claiming via `OFLOW_BOARD=gitlab`
- **Run reporting with heartbeat**: Added heartbeat, estimates, artifacts, and step events to run reporting for better observability
- **OpencodeAdapter**: New adapter with exit-code tracking and tests
- **Agent prompts extracted**: Inline agent prompts moved to standalone agent files for maintainability
- **Daemon status command fixed**: Wired `writeSessionsJson` into the daemon loop so `oflow status` correctly shows active sessions
- **Removed process.exit from validateArtifactCmd**: Replaced with thrown errors, making the function unit-testable; added 8 unit tests
- **Deduplicated getCurrentTaskId**: Extracted helper moved to `src/cli/utils.ts`
