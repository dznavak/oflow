# Release Notes

## Recent Changes

- **Per-step timeout and hang detection**: Added configurable per-step timeout (default 900s) to the daemon loop; steps that exceed the limit are killed via SIGTERM and marked as timed-out, freeing the session slot for subsequent runs.
- **Event-driven adapter resolution**: Replaced `setInterval` polling in `ClaudeCodeAdapter` and `OpencodeAdapter` with event-driven resolution on child process `close`, eliminating potential hangs in `waitForCompletion`.
- **Shared `getCurrentTaskId` utility**: Extracted duplicate function from `state.ts` and `validate.ts` into `src/cli/utils.ts` to eliminate code duplication.
- **`validateArtifactCmd` refactor**: Replaced `process.exit()` calls with thrown `Error`s, making the function unit-testable; added 8 unit tests covering all code paths.
- **Status command fix**: Wired `writeSessionsJson` into the daemon loop so `oflow status` correctly shows active sessions instead of always reporting none.
- **GitLab board adapter**: Implemented `GitLabBoardAdapter` with full `BoardAdapter` interface, supporting label-based workflow management and task listing/claiming/updating.
- **Opencode adapter**: Added `OpencodeAdapter` with exit code tracking and tests.
- **Run reporting**: Added heartbeat, estimates, artifacts, and step events to run reporting for better observability.
- **Multiagent code review skill**: Added parallel specialist agent skill for code review with assembly step.
- **Dev-workflow git setup**: Moved git branch setup to Step 0 to ensure a clean branch before any exploration or code changes.
