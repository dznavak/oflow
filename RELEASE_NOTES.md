# Release Notes

## Recent Changes

- **E2E verification pipeline**: Added `scripts/e2e.sh` and `e2e-verification` step skill that validates the full oflow pipeline end-to-end before any PR is opened — creates a real GitHub issue, spawns the daemon, and polls for a resulting PR
- **Release-notes workflow**: New lightweight `release-notes` skill used as the e2e test target — summarises recent commits and opens a PR adding `RELEASE_NOTES.md` in ~90 seconds
- **Daemon spawn & logging fixes**: Fixed Claude Code adapter to spawn via stdin, stream stdout/stderr to terminal in real-time with `[task-X]` prefix, and track exit codes accurately
- **Custom label filtering**: Added `--label` option to `oflow board list`, `board pick`, and `run` commands so tasks can be filtered by a secondary label beyond the base `taskLabel`
- **Skills restructured to folder format**: Each skill is now a folder with `SKILL.md` inside, matching the agentskills.io open format; Claude Code discovers them natively under `.claude/skills/`
- **Config & test isolation**: Moved `dotenv.config()` out of `loadConfig()` into the CLI entry point so unit tests are not polluted by `.env`; excluded `.worktrees/` from vitest discovery
- **CI fixes**: Resolved ESLint errors (node globals, unused vars), added `package-lock.json` for `npm ci`, and fixed missing `number` field in `TaskContext`
- **Core milestones landed**: GitHub board adapter (M1), state manager + artifact schemas (M2), dev-workflow skills (M3), agent adapters + daemon scheduler/poller + full CLI wiring (M4)
