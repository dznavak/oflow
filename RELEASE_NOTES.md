# Release Notes

## Recent Changes

- **Label-based task filtering**: `oflow board list`, `board pick`, and `run` now accept `--label <label>` to filter tasks by a custom GitHub label
- **Improved Claude spawning**: Claude Code adapter now spawns via stdin (fixes YAML frontmatter parsing issues), with real-time log streaming and `[task-X]` prefixed output
- **Daemon logging improvements**: Startup now logs board/agent/label/slot config; spawn events show PID and log path; completion shows duration; failures show log path
- **Fixed spawn-log loop**: Daemon poller no longer re-logs already-active sessions as "started" on every poll iteration
- **Skills restructured as folders**: Each skill is now a `<name>/SKILL.md` folder matching the agentskills.io open format; Claude Code discovers them natively from `.claude/skills/`
- **CI fixes**: Resolved ESLint errors (node globals, unused vars), added `package-lock.json` for `npm ci`, excluded `.worktrees` from vitest discovery
- **M4 — Full daemon + CLI wiring**: Agent adapters (ClaudeCode, Opencode), scheduler slot management, poller, spawner, `run`/`status`/`state`/`validate` CLI subcommands, and 14 new tests
- **M3 — Workflow skills**: Dev-workflow pipeline skills for exploration, planning, implementation, and review steps
- **M2 — State manager**: Artifact schemas and validator with field-level error reporting
- **M1 — GitHub board adapter**: Board CLI commands (`list`, `pick`, `claim`, `update`)
