---
name: release-notes
description: Summarise recent git changes and open a PR adding RELEASE_NOTES.md — used as the e2e verification workflow
---

You are executing the release-notes workflow for an oflow task.

## Setup
Read the task context:
```bash
cat .oflow/runs/$(cat .oflow/current)/task-context.json
```
Note the `id` (issue number) and `repoPath`.

## Steps

### 1. Gather recent changes
```bash
git log --oneline --no-merges -20
git log --format="%s%n%b" --no-merges -10
```

### 2. Create a branch
```bash
git checkout -b release-notes/$(cat .oflow/current)
```

### 3. Write RELEASE_NOTES.md
Based on the git log output, write a concise `RELEASE_NOTES.md` at the repo root.

Format:
```markdown
# Release Notes

## Recent Changes

- <bullet summarising each meaningful commit group>

_Generated by oflow e2e verification._
```

### 4. Commit and push
```bash
git add RELEASE_NOTES.md
git commit -m "docs: add release notes"
git push -u origin HEAD
```

### 5. Open the PR
```bash
gh pr create \
  --title "docs: release notes" \
  --body "$(cat <<'PRBODY'
## Summary
- Adds RELEASE_NOTES.md summarising recent repository changes

## Related Issue
Closes #<ISSUE_NUMBER>

🤖 Generated by oflow release-notes workflow (e2e verification)
PRBODY
)"
```
Capture the PR URL.

### 6. Update the task
```bash
oflow board update <TASK_ID> --status done --message "PR opened: <PR_URL>"
```

## Notes
- Keep the release notes short — 5–10 bullet points is enough
- The PR body must contain `Closes #<issue_number>` so the e2e script can find it
- Do not run tests or create artifacts beyond what is listed here
