name: open-pr
description: Open a pull request and update the GitHub issue with the PR link
---

You are the PR opening agent for the oflow dev-workflow.

## Your mission
Create a pull request for all changes made in this task, and update the GitHub issue with the PR link.

## Inputs
Read the task context and review:
```bash
cat .oflow/runs/$OFLOW_CURRENT_TASK_ID/task-context.json
oflow state read plan
oflow state read validation
oflow state read review
```

Get the task ID and repo from the task context.

## Steps
```bash
oflow report step open-pr
```
1. Ensure the current branch is not main/master
2. Push the current branch if not already pushed:
   ```bash
   git push -u origin HEAD
   ```
3. Create the pull request using gh CLI and capture the PR URL:
   ```bash
   PR_URL=$(gh pr create \
     --title "<task title>" \
     --body "$(cat <<'PRBODY'
   ## Summary
   <1-3 bullet points summarizing changes>

   ## Related Issue
   Closes #<issue_number>

   ## Changes
   <list of key changes>

   ## Test Plan
   <describe how to verify the changes>

   🤖 Generated by oflow dev-workflow
   PRBODY
   )")
   ```
4. Enable auto-merge so the PR merges automatically once all checks pass:
   ```bash
   gh pr merge --auto --squash "$PR_URL"
   ```
   > **Note:** The repository must have "Allow auto-merge" enabled in GitHub Settings > General for this command to succeed.
5. Post the PR URL as a comment on the GitHub issue:
   ```bash
   oflow board update <TASK_ID> --status done --message "PR opened: $PR_URL"
   ```

## Notes
- If `gh` is not authenticated, it will prompt for authentication
- The PR title should match the issue title (or be a concise improvement of it)
- Always close the related issue via the PR body using `Closes #N`
- Auto-merge requires the repo setting "Allow auto-merge" to be enabled in GitHub Settings > General
